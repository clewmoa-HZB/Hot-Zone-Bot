# Utilisation et sécurité du projet

## Versions supportées

Les versions des modules utilisés sont visibles dans le fichier "requirements.txt" situé à la racine du projet.
Ce sont ces modules avec la version précisée qui doivent être utiliser.

Pour ajouter les modules à votre interpreteur Python, veuillez vous référer au fichier [README.md](https://github.com/clewmoa-HZB/Hot-Zone-Bot/blob/main/README.md)

## Signaler du code vulnérable

### Signaler une vulnérabilité

Vous pouvez signaler une vulnérabilité dans le code en créant une "issue" avec le tag/label "vulnerability"
et en décrivant la vulnérabilité trouvée

> Une réponse devrait être envoyée au plus vite. Notamment le week-end si ce n'est la semaine

Une vulnérabilité signalée sera étudiée au plus vite puis acceptée ou rejetée selon la gravité potentielle
des dégâts que cette vulnérabilité peut causer et selon l'accessibilité de celle-ci.
Si la vulnérabilité n'est pas accessible à beaucoup de monde, la résolution de celle-ci pourra alors prendre plus de temps

### Signaler une vulnérabilité grave

###### Du code pouvant mettre en danger vos données liées au projet par exemple

Si tel est le cas, faites exactement la même chose que pour une vulnérabilité classique
mais ajoutez cette fois-ci les labels `vulnerability` et `security risk`

> Vous pouvez aussi contacter @clewmoa-HZB via le mail GitHub ou sur Discord,
ou en ouvrant un ticket sur le support du [serveur Discord](https://discord.gg/SC2X7yWU6Y)
